Executive Summary

Project Hyena is an event-driven network monitoring and incident management platform specifically designed for the hospitality industry. Built to address the unique connectivity challenges faced by South African hotel chains, Hyena transforms reactive firefighting into proactive operational excellence through real-time monitoring, intelligent workflow automation, and multi-role collaboration.

This prototype demonstrates a comprehensive white-labeled SaaS platform serving multiple competing hotel chains while maintaining strict data isolation. The system enables guest self-service issue reporting, property-level incident management, regional multi-property oversight, field technician coordination, and platform-wide administration—all through purpose-built, role-specific interfaces.

Business Problem Statement

Modern hotels depend on reliable network connectivity for guest satisfaction, but traditional IT support models are reactive, opaque, and inefficient. Guests experiencing WiFi or streaming issues must call reception, wait for technician response, and remain uninformed about resolution progress. Property managers lack visibility into incident patterns, technicians waste time on coordination overhead, and regional directors cannot optimize resources across their portfolio.

The result: guest frustration, operational inefficiency, missed revenue opportunities, and competitive disadvantage in an increasingly connected hospitality market.

Hyena's Solution: Event-Driven Transparency + Role-Specific Intelligence

Hyena replaces fragmented support processes with a unified incident lifecycle platform where every stakeholder—from guest to CEO—has appropriate visibility and control. Real-time status updates, automated workflow routing, collaborative timelines, and analytics dashboards transform network support from cost center to operational differentiator.

Current prototype delivers immediate value. AI enhancement unlocks transformational competitive advantage.

System Architecture Overview

Multi-Tenant White-Label Design

Hyena operates as a SaaS platform serving multiple hotel chain organizations (Protea Hotels, City Lodge, etc.) with complete brand customization and strict data isolation. Each organization sees only their properties, users, and incidents within a fully branded experience matching their corporate identity.

Role-Based Interface Architecture

Six purpose-built interfaces optimize workflows for distinct user roles:

1. Guest Portal

Users: Hotel guests

Purpose: Self-service issue reporting, troubleshooting guides, status tracking

Key Value: Empowerment, transparency, reduced support call volume

2. Hotel Manager Dashboard

Users: Property managers (single hotel)

Purpose: Operational command center for property-level incident management

Key Value: Real-time oversight, actionable analytics, resource optimization

3. Regional Manager Dashboard

Users: Regional directors (3-15 properties)

Purpose: Multi-property performance comparison and resource allocation

Key Value: Portfolio intelligence, best practice propagation, strategic planning

4. Technician App

Users: Field support technicians

Purpose: Mobile-optimized work queue, incident resolution, team collaboration

Key Value: Efficiency, communication, knowledge capture

5. Admin Center

Users: Hyena platform administrators

Purpose: Platform-wide organization, user, and property management

Key Value: Scalable multi-tenant operations, compliance, business intelligence

6. Event Simulator

Users: Administrators, developers, trainers

Purpose: Testing, training, demonstration scenario generation

Key Value: Risk-free validation, staff onboarding, sales enablement

Technical Stack

  • Frontend: React 18+ with TypeScript, Radix UI, shadcn/ui, Tailwind CSS
  • Backend: Node.js with Express, RESTful API design
  • Database: PostgreSQL (Neon serverless) with Drizzle ORM
  • State Management: TanStack Query (React Query v5) for server state
  • Authentication: Role-based access control with session management
  • Routing: Wouter for client-side navigation
  • Build System: Vite for optimal development and production builds

Core Functional Capabilities

Incident Lifecycle Management

Complete incident workflow from initial report through resolution with collaborative timeline tracking:

  • Creation: Guest portal submission, manager manual entry, automated monitoring (future)
  • Triage: Automatic or manual categorization, priority assignment, technician routing
  • Active Work: Status progression (New → Assigned → In Progress → On Hold → Resolved)
  • Collaboration: Comment system enabling manager/technician communication
  • Actions: Cancel, reassign, escalate, request information, change priority
  • Resolution: Technician resolution with optional root cause documentation
  • Audit Trail: Complete timeline of all actions, actors, timestamps for compliance

Alert vs. Incident Separation

System-wide classification distinguishing actionable problems from informational status updates:

  • Incidents (actionable): Reactive failures, outages, security issues requiring technician remediation
  • Alerts (informational): External conditions, scheduled maintenance, proactive notifications
  • Workflow Impact: Incidents route to technician queues; alerts display in status dashboards
  • Metrics Separation: Dashboard KPIs filter to incidents only for accurate operational metrics

White-Label Branding

Organization-specific customization enabling platform to serve competing hotel chains:

  • Visual Branding: Custom logos, color schemes, typography per organization
  • Multi-Tenant Isolation: Strict data separation between organizations
  • Scoped Access: Users see only their organization's properties and data
  • Branded Exports: Reports and communications match organization branding

South African Localization

Platform specifically adapted for SA hospitality market requirements:

  • Load Shedding Integration: Eskom API integration for proactive impact communication
  • ISP Awareness: SA provider patterns (Telkom, Vodacom, MTN, Rain)
  • Currency & Formats: ZAR, SA date/time formats, metric units
  • Event Categories: SA-specific (Load Shedding, ISP Outage, Weather Impact)
  • Language Support: English with multi-language groundwork (Afrikaans, Zulu)
  • POPIA Compliance: Data privacy aligned with Protection of Personal Information Act
