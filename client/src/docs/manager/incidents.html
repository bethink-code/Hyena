Incident Queue - Functional Specification

Overview

The Incident Queue page displays a filterable, searchable list of network incidents. It serves as the central hub for incident management, allowing managers to view, search, filter, and take action on incidents across their assigned properties.

Page Structure

1. Filter Description Header

Shows currently active filters with visual hierarchy:

  • Property Name: Displayed in orange badge for maximum prominence
  • Status/Priority: Shown as regular text
  • Clear Filters: X icon button appears when filters are active

2. Search and View Controls

  • Search Bar: Real-time search across incident titles, descriptions, and locations
  • View Switcher: Toggle between Cards, Table, and Grid views
  • Status Filter: Dropdown to filter by incident status
  • Priority Filter: Dropdown to filter by priority level

3. Incident List

Displays incidents in selected view mode (Cards/Table/Grid). Each incident shows:

  • Title and description
  • Status badge (color-coded)
  • Priority indicator
  • Property name
  • Assigned technician (if any)
  • Location information
  • Timestamp

Filtering Behavior

URL-Based Filters

Filters can be set via URL parameters for shareable links:

  • ?status=in_progress - Shows only in-progress incidents
  • ?priority=critical - Shows only critical incidents
  • ?propertyId=1 - Shows incidents for specific property

Default Behavior

Without explicit filters, the page shows only active incidents (excludes terminal statuses: resolved, cancelled, duplicate).

Clear Filters

Clicking the X icon resets all active filters simultaneously and shows all active incidents.

Incident Detail Panel

Clicking any incident card/row opens a sliding panel from the right with three tabs:

1. Details Tab

  • Full incident information
  • Property and location details
  • Category and event type
  • Metadata (for SA-specific incidents)

2. Activity Tab

  • Timeline of all incident actions
  • Status changes with timestamps
  • Assignment history
  • Priority escalations/de-escalations
  • Information requests and responses

3. Actions Tab

Available actions (for active incidents only):

  • Assign Technician: Assign or reassign to a technician
  • Change Priority: Escalate or de-escalate priority level
  • Request Info: Log information requests
  • Put On Hold: Pause work with reason and optional resume date
  • Cancel Incident: Mark as cancelled with required reason
  • Resolve: Mark incident as resolved

Note: Actions are hidden for terminal statuses (cancelled, resolved, duplicate).

Status Lifecycle

Incidents progress through these states:

  • New: Just created, not yet assigned
  • Assigned: Assigned to technician, work not started
  • In Progress: Technician actively working on it
  • On Hold: Work paused (waiting for parts, vendor, etc.)
  • Resolved: Work completed successfully (terminal)
  • Cancelled: Incident cancelled with reason (terminal)
  • Duplicate: Marked as duplicate of another incident (terminal)

Keyboard Shortcuts

  • Esc - Close detail panel
  • / - Focus search input
