Technician Work Queue - Functional Specification

Overview

The Technician Work Queue displays incidents assigned to the logged-in technician. It serves as the primary interface for technicians to view their workload, prioritize tasks, and update incident progress.

Dashboard Metrics

Summary Cards

Four key metrics displayed at the top:

  • My Queue: All active work items (new, assigned, in_progress, on_hold)
  • In Progress: Incidents currently being worked on (status='in_progress')
  • Completed Today: Incidents resolved today
  • Critical: Active incidents with critical priority requiring immediate attention

Clicking any metric navigates to filtered incident list.

Work Queue Features

1. Filtered Incident List

Shows incidents relevant to the technician with URL-based filtering:

  • ?status=in_progress - Shows work in progress
  • ?priority=critical - Shows critical priority items
  • ?propertyId=1 - Shows work for specific property

2. Search and Navigation

  • Real-time search across incident details
  • View switcher (Cards/Table/Grid)
  • Back to Dashboard button for easy navigation

3. Incident Actions

Technicians can perform these actions on assigned incidents:

  • Start Work: Change status from 'assigned' to 'in_progress'
  • Request Info: Request additional information from reporter
  • Put On Hold: Pause work (waiting for parts, vendor response, etc.)
  • Resolve: Mark incident as completed
  • Add Notes: Log progress updates and technical details

Incident Detail Panel

Three-tab interface for managing incidents:

1. Details Tab

  • Full incident description
  • Property and location information
  • Category and priority
  • Metadata (ISP details, load shedding info, weather conditions)
  • Contact information for reporter

2. Activity Tab

  • Complete timeline of incident history
  • Status changes with timestamps
  • Assignment history
  • Information requests and responses
  • Notes added by other technicians or managers

3. Actions Tab

  • Available actions based on current status
  • Forms for updating incident status
  • Required fields for each action (e.g., reason for hold)

Work Prioritization

Incidents are color-coded by priority:

  • Critical (Red): Network outages, critical system failures
  • High (Orange): Service degradation affecting multiple users
  • Medium (Yellow): Individual user issues
  • Low (Blue): Minor issues, non-urgent requests

Technicians should prioritize critical and high-priority incidents first.

Status Updates

Technicians update incident status as work progresses:

  1. Assigned: Incident assigned to technician
  2. In Progress: Technician actively working on resolution
  3. On Hold: Work paused (requires manager approval to resume)
  4. Resolved: Issue fixed and verified

Property Scope

Technicians see incidents from all assigned properties. Property assignments are managed by administrators through the Users & Roles interface.

Note: Once authentication is implemented, the queue will automatically filter to the logged-in technician's assigned incidents.

Mobile Optimization

The technician interface is optimized for mobile/tablet use:

  • Touch-friendly buttons and controls
  • Responsive card layout
  • Quick action buttons
  • Offline capability (future enhancement)
